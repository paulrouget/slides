DevTools UX.

An experimental design.

3 rules

Things we are trying to fix with this design.

1. Don't lose the user

Too many buttons/tabs/popups/windows is bad. Minimize the number of clicks. Share designs patterns between tools. We need to be consistent (between tools & Firefox).

2. Avoid content invasion

The content is sacred. Hiding 30% of the web page is bad. Can we avoid that?

3. Customizable

We can't assume anything. Different use cases. We don't know a lot about developers habits. The developer needs to feel confortable.

a debugging story

  "I want to debug"
      -> Reach the developer-tools entry point
         -> "I want to do something with this page"
           -> Launch a Page Centric tool
         -> "I want to work on a node"
           -> Launch the inspector. Select a node
             -> "I want to do something with this node"
               -> Launch a Node Centric tool
